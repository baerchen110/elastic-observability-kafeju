# Feedback - Challenge 02 (Explore Data and Dissect a Tool)

- In Step 1, it would help to provide an example answer after each question so learners can validate their interpretation.
- From the table alone, it is hard to infer the expected insights; customers may struggle to understand the data model and answer correctly without additional guidance.
- For Steps 2 and 3, consider showing both paths: Kibana UI as the easy path and terminal/API as the advanced path.
- If both paths are included, explain when to use each one and keep expected outcomes aligned between them.
- Across all steps, after the learner action and a short pause to think, include a brief explanation of what they should notice in the result and why it matters.
- Example for clearer coaching language: instead of only saying "Compare the raw query results with the agent's narrative answer," explicitly point out what is different in each output and why (e.g., "Here the tool returns structured rows; there the agent summarizes implications. This difference exists because tools provide raw data while the model adds interpretation.").
- The challenge is named "ML anomaly detection," but it does not clearly explain where ML has been run on the data, what artifact/output comes from ML, and what additional insight ML provides compared to raw querying; this should be explained in more detail.
